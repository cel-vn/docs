# AI Content Generation Rules for Fumadocs

This document outlines the comprehensive rules and guidelines for AI-generated content within the CEL Developer documentation platform.

## üéØ Core Principles

### 1. Content Quality Standards

#### Accuracy Requirements
- **Technical Precision** - All code examples must be syntactically correct and functional
- **Version Consistency** - Content must align with the current software versions
- **Fact Verification** - Claims and statements should be verifiable against source materials
- **Link Validation** - All external references must be active and relevant

#### Clarity Guidelines
- **Simple Language** - Use accessible terminology with technical jargon explanations
- **Logical Flow** - Organize content in a logical, progressive manner
- **Clear Examples** - Provide concrete, real-world examples for abstract concepts
- **Visual Hierarchy** - Use headings, lists, and formatting for scanability

### 2. Documentation Structure Rules

#### Semantic Organization
```markdown
# Main Topic (H1) - One per document
## Major Sections (H2) - Primary content divisions
### Subsections (H3) - Detailed breakdowns
#### Details (H4) - Specific implementations
##### Examples (H5) - Code samples and use cases
###### Notes (H6) - Additional information
```

#### Content Patterns
- **Introduction** - Overview and objectives
- **Prerequisites** - Required knowledge and setup
- **Step-by-Step** - Numbered procedures with validation
- **Examples** - Working code with explanations
- **Troubleshooting** - Common issues and solutions
- **Next Steps** - Related topics and advanced concepts

## üìù Content Generation Workflows

### 1. Automated Documentation Pipeline

#### Code Analysis Phase
1. **AST Parsing** - Extract component structure and types
2. **Dependency Mapping** - Identify imports and relationships
3. **Usage Pattern Detection** - Find common implementation patterns
4. **Example Extraction** - Collect existing usage examples

#### Content Generation Phase
1. **Template Selection** - Choose appropriate documentation template
2. **Content Assembly** - Combine analysis with generated explanations
3. **Example Creation** - Generate working code examples
4. **Cross-Reference Building** - Link related components and concepts

#### Quality Assurance Phase
1. **Syntax Validation** - Verify all code examples compile
2. **Link Checking** - Ensure all references are valid
3. **Accessibility Review** - Check for WCAG compliance
4. **Performance Analysis** - Validate load times and optimization

### 2. Human-AI Collaboration Rules

#### AI Responsibilities
- **Content Generation** - Create initial drafts and structures
- **Code Analysis** - Extract technical information from source
- **Pattern Recognition** - Identify documentation opportunities
- **Consistency Checking** - Maintain style and format standards

#### Human Responsibilities
- **Strategic Direction** - Define documentation goals and audience
- **Quality Review** - Validate accuracy and relevance
- **Creative Input** - Add context, insights, and narrative
- **Final Approval** - Authorize publication of generated content
